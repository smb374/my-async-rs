Module my_async::schedulers

source ·
Expand description

Scheduler trait an various default implementation.

The module defines and implements various crutial parts for multi_thread::Executor to work:

  • Scheduler trait: Defines the behaviour of a schduler should have for the multi-threaded executor to work.
  • JoinHandle: The future task handle for async and non-blocking joining a futture.
  • FutureJoin: Future implementation for async join.
  • Spawner: Multi-threaded future task spawner that is initialized during scheduler initialization.
  • spawn() and shutdown(): Multi-threaded version of the single-threaded version. Need Spawner to be initialized to work.
  • ScheduleMessage: Interthread message to communicate with Scheduler.
