Multi-threaded executor.

This multi-threaded executor needs a Scheduler for thread creation and task scheduling.

The executor itself on main thread won’t do any work since it’s main job is to receive scheduling message from the spawner. There is also a IO polling thread for event notification. The worker threads will be (n - 2) threads where n is the cpu number.

For more information, see documentation for schedulers.
